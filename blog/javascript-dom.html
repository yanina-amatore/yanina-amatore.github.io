An analogy to describe JavaScript and its relationship to HTML and CSS.

I love action sci/fi movies, so to explain how JavaScript relates to HTML and CSS, I chose an analogy about how these films making processes.

At the beginning of the movie production, writers create a story in a documented script which describes all the characters and scenes to be performed. This would be the HTML part of the movie, let's say. If the actors performed just what was written in that text without any extra aid, it would look more like a minimalistic theatre play than an actual action movie.

But to properly tell a story, we need more context that could be provided by adding styling. This is when the director would hire CSS as a designer. It would be in charge of how the elements (characters, props and film sets) will look on the screen. It can manipulate and style them in fancy costumes, add make-up and even change the backgrounds for different scenes.

However, It cannot be a sci-fi movie if no special effects are involved. In this analogy, JS would be the post-production movie team in charge of adding CGI (computer-generated imagery) and other special effects to our movie. For example, they could take an element of the film like the actor Mark Ruffalo playing Hulk and make him look big, green and angry or make Thor fly onscreen when spinning his hammer fast. Likewise, JS working along HTML and CSS is used to dynamically change a page's behaviour, display and interactions.

Hope you’ve enjoyed the analogy.

Explain control flow and loops using an example process from everyday life, for example, 'waking up' or 'brushing your teeth' (but not those).

Control flow is how your computer reads and runs code from top to bottom. It starts from the first line and ends at the last line unless it hits any statement that changes the program's control flow (loops, conditionals, or functions). These mentioned loops repeatedly run a code block until a specific condition is met. Only then the computer will jump to read the following line. To understand it better, let's check an everyday example.

These days that I'm doing dev academy, my partner is the designated house cook. To make life easy, we decided to do Hello Fresh( a meal delivery service with pre-packed ingredients and portions). To successfully cook their meals, he must read and follow the recipes instructions from top to bottom like using control flow, or we might end up with a failed dinner.

If we Apply the control flow and loops concepts, the recipe steps will look like this:

QUICK & EASY RED LENTIL DAHL

  1. chop onions
  2. place a pan over medium heat.
  3. saute the chopped onions in olive oil and stir them for 5 minutes (loop)

    4. // --- the line above is a loop in which zack has to complete the action until the condition of 5 min is met -- //

  4. add chopped garlic and ginger
  5. cook for 1 min
  6. add the garam masala, turmeric and red pepper flakes to the pan
  7. stir the onion and spice mixture for 2 minutes
  8. if the mix is too dry, add a few tablespoons of water (conditional)

    9. // -- the line above has a conditional. When zack runs into it, he has to determine if the condition is met to continue with the recipe or else he has to "add a few spoons of water -- //

  9. add the dried lentils
  10. add canned tomatoes and their juices
  11. add coconut milk and vegetable broth to the pan
  12. stir well and turn the heat to high
  13. bring to a boil
  14. then lower heat and simmer for about 15 minutes, until the lentils are cooked and soft.

    15. // --- the line above is a loop in which the mixture has to simmer for 15 min, and once the condition is met, the loop ends, and zack can go to the next step -- //

  15. stir occasionally
  16. squeeze the lemon juice into the pan
  17. stir in the spinach as well until wilted
  18. Add salt to taste
  19. Serve with brown or white rice and Vegan Naan. Enjoy!

Explain what functions are and why they are helpful.

A function in JavaScript is similar to a procedure—a set of statements that have been grouped together because they perform a specific task. The statements in a function are not always executed when a page is load, so functions also offer a way to store the steps needed to achieve a task. When the script repeat the same task, you can reuse the function (rather than repeating the same set of statements). You need to provide a name for your function in order to call that name if you want it to perform a task later.
