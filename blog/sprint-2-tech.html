Inline vs Inline-Block Display in CSS

30 june 2022


introduction

In CSS, the display property specifies the behaviour of an element. It’s a must-know for controlling the layout of an element. In this blog entry, we’ll learn the difference between the two display options: Inline and Inline-block.

Inline

This one displays the element on the same line. In other words, inline elements do NOT start on a new line and only take up as much width as their content. So, if you try to set any width and height, it will have NO effects.

Some elements have a default inline property for example:

  • span
  • a
  • img

    • And most of the formatting tags are also are inherently inline

    • em
      • gir
    • strong
    • i

    Inline-block

    Displays an element as an inline-level block container. You CAN set height and width values. It’s essentially the same thing as inline, except that you can set height and width values.

    In summary: Compared to display: inline, the major difference is that inline-block allows setting a width and height on the element. Also, with display: inline, top and bottom margins & paddings are not respected, and with display: inline-block they are.
