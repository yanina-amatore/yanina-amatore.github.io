Inline vs Inline-Block Display in CSS

30 june 2022


introduction

In CSS, the display property specifies the behaviour of an element or tag. It’s a must-know for controlling the layout of an element. In this blog entry, we’ll learn the difference between the two display options: Inline and Inline-block.

Inline

This one displays the element on the same line. In other words, inline elements do NOT start on a new line and only take up as much width as their content. So, if you try to set any width and height, it will have NO effects.

It's good to have in mind that some elements have a default inline property for example:

  • span
  • a
  • img

    • And most of the formatting tags are also are inherently inline

    • em
    • strong
    • i

    Inline-block

    When we displays an element as an inline-level block it looks like a container. The great news about it is that You CAN now set height and width values. It’s essentially the same thing as inline, except that you can set height and width values.

    Check out a comparison between the two of them in action. In this example, I chose a DIV because it's inherently a BLOCK element and I wanted to play with other display options to show you how it behaves.

    In summary

    The major difference between Inline and inline-block is that the last one allows us to set a width and height on the element. Also, with display: inline, top and bottom margins & paddings are not respected, and with display: inline-block they are.
